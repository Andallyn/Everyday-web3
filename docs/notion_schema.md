# Notion Editorial Desk Schema

Create five Notion databases and use the source CSV fields as the import baseline.

## Sources

| Property | Type | Notes |
| --- | --- | --- |
| Title | Title | Source headline or working idea |
| URL | URL | Original source link |
| Summary | Text | What happened and why it matters |
| Category | Select | Everyday Web3 bucket |
| Source Type | Select | post, article, event, meetup, conference, blog, press |
| Location | Text | City, country, online, remote |
| Event Date | Date | Event or announcement date |
| Tags | Multi-select | Topic tags |
| People | Multi-select | Creators, founders, companies |
| Priority | Number | 1-5 |
| Status | Select | new, reviewed, selected, drafted, scheduled, published, archived |
| Assigned To | Person | Operator/owner |
| Publish Date | Date | Intended publication date |
| Platform Status | Text | Per-channel notes |
| Performance Notes | Text | Qualitative results |
| Repurpose | Checkbox | Reuse this idea later |

## Content Drafts

Track platform, body, CTA, hashtags, source relation, status, scheduled time, and media notes.

## Publishing Calendar

Track publish date, platform, campaign, status, source relation, and final URL.

## Companies

Track company name, category, website, X handle, source URL, region, tags, priority, and notes.

## Performance

Track impressions, engagements, clicks, saves, comments, winning hook, and recommended next action.
